EXHIBITION BUILDER

EXHIBITION BUILDER

Exhibition Builder is built mainly using client side programming, HTML5 and Javascript. It started with an idea to build a tool for creating an online presentation using the media available in EUscreen and quickly escalated into a separate product. The development process started with several workshops involving the target end user and after many hours of brainstorming and meeting, we set out to create the wireframe and after the wireframe is done, the design phase started and followed by the development phase. The application should be straightforward, easy to use, focused on usability which are the three main guidelines when the UI/UX is created.

THE PROCESS

  • Workshops

    It all started with trying to get better understanding of what the target user needs, workshops is a great forum where the user needs can be voiced and understand.

  • Wireframing

    Wireframe is a great tool and the first step to focus on creating a great user experience and mapping all the functionalities in the interface.

  • UI Design

    Great UI are more than just pixels with pretty colors, they are unified systems of visuals and interactions that solve complex problems with empathy for the end-user.

  • Development

    The development phase is the phase where all ideas, functionalities and design fused together into a reality. Ideas are cheap but precise executions is what makes a great product.

PLATFORM

Exhibition Builder is running atop a solid cloud-based services which uses REST foundation for data transfer. The app serves as a front-end to these cloud-based services while the rest is running in the cloud (databases, sessions, uploading, etc). A customized object oriented javascript platform is built to maximize the integration between the front-end and the services.
